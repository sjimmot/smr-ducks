# Repositorio del grupo SMR-Ducks
• En este repositorio de GitHub tuvimos nuestra primera toma de contacto
a nivel colaborativo con el sistema de control de versiones "git".

## Resumen y objetivos
• El objetivo era poner en práctica nuestra capacidad organizativa, ser
capaces de generar y utilizar un token o par de claves pública y privada,
además de realizar correctamente un commit y push al repositorio.

• Además, el repositorio debería tener la siguiente estructura:

~~~
[-] smr-ducks
 |
 |- [-] Nombre de alumno
 |   |- archivodetexto.txt   <--- Archivo individual con una reflexión.
 |
 | (...)
 |
 |- alumnos.txt   <--- Archivo colectivo donde todos deben añadir su nombre.
 |- README.md   <--- Esta descripción de repositorio.
~~~

• Iniciamos creando carpetas individuales en las que añadimos una explicación del porqué nos gusta la informática.
También actualizamos el "alumnos.txt" con nuestros nombres individualmente en cada "commit" de cada alumno.

• Por último actualizamos el archivo "README.md" incluyendo una pequeña descripción, resumen y pasos a seguir durante la creación del repositorio, así como la estructura del mismo.
